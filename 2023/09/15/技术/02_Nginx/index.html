Nginx（轻量级web服务器）

Tomcat服务器
Nginx服务器

tomcat

  • java语言开发
  • 支持servlet规范（直接java web 应用的部署）

Nginx服务器

  • 基于Http协议，不部署任何web应用，代理服务器，反向代理

  • c语言开发

  • 不支持任何web应用

Tomcat服务器集群

前端发请求直接发给Nginx——》Nginx再转发给tomcat服务器集群

两大功能

反向代理

翻墙——》正向代理服务器

  • 代理的是发起请求的乙方

反向代理服务器

  • Nginx代理接收处理请求的一方服务器——》Nginx代理tomcat服务器集群

高性能的静态Web资源服务

  • 可以把静态资源放在Nginx中

Nginx

多进程（多线程）

  • 高并发、高性能
  • 扩展性好
  • 异步非阻塞的事件驱动模型（io比较好）
  • 高可靠性
    • 多个Worker进程（容错性高）
    • 修改Nginx的部署文件（不重启也可以生效）

nginx的路径

配置文件路径

/etc/nginx/nginx.conf

命令路径

/usr/sbin/nginx

日志路径

/var/log/nginx

  • nginx常用命令

    • 找到Nginx执行命令，以Ubuntu18.04为例

    cd /usr/bin

    sudo ./nginx #启动

    sudo ./nginx -s stop #关闭

    sudo ./nginx -s reload #重启

  • 配置文件配置

    • 全局配置

      • worker_processes auto
        工作进程设置auto自动（有几核，创建几个）
        可手动指定，核数2n左右
      • include 路径（其它文件配置，引入进来）
        主进程文件存在pid配置的路径

    • 事件模型配置

      • /# multi_accept on;
        请求分配方式（是否允许一次处理多个请求）

    • Http协议模块配置⭐

      • 配置多个虚拟server(每个虚拟serve代理不同的目标)
        • 一个虚拟serve——》多个location(路由映射规则)(定义转发规则，一条转发和规则，就是一个location

      0拷贝，

      sendfile on;

      高效的io (开启 sendfile on，数据从内核缓冲区直接拷贝到Socket缓冲区，省略内核——》操作系统——》socket缓冲区的过程）

      tcp_nopush on;

      不会立马把数据发送出去，会用一个缓冲区，先存起来，等到达一定的数量之后，再封装（加数据头）——》节省带宽，提升有效载荷的效率

      tcp_nodelay on；

      无延迟立即发送

      要不要在请求头带版本号

      server_tokens of；

      响应头要不要带虚拟servie的名字

      server_name_in_redirect off;

      默认数据类型

      default_type application/octet-stream;

      优先服务器的加密方式

      ssl_protocols TLSv1 TLSv1.1 TLSv1.2 TLSv1.3; # Dropping SSLv3, ref: POODLE
      ssl_prefer_server_ciphers on; 优先服务器的加密方式

      压缩（nginx向浏览器响应时压缩）

      gzip on;

  • Nginx基本的反向代理功能

  • 负载均衡策略

    • 轮询策略（默认）

    • 加权策略

    • ip_hash策略（散列效果差）
      发起请求的ip地址映射到tomcat服务器

    • least_conn 最少连接数策略
      nginx可以统计出当前每个Tomcat并发连接数

    • fair（第三发）最少响应时间

      nginx统计Tomcat平均响应时间

    • url_hash（第三发）（散列效果更好）

      同一台主机访问请求，可以被不同的tomcat处理（参数值）
