session和cookie的区别

发表于 2023-10-07 更新于 2023-09-08 分类于 每日一题 阅读次数：
未来的路就在脚下，不要悲伤不要害怕，充满信心期盼着明天。

梳理主线：

  1. http是无状态的协议
  2. 因为它的无状态性，所以无法维持会话，因为这个原因，我们引入了会话技术
  3. 简单来说cookie和session，是存储了一些信息，可以识别到发送的请求来自谁。
  4. 他们二者的区别在于，cookie是客户端技术，它把信息以键值对的形式存储再浏览器，类型是string类型的，在cookie的使用，session是服务端技术，所以存储信息的大小，用session存储的信息格式，是不是敏感信息，能不能够跨应用

cookie和session的区别

谈到cookie和session,我们不得不说一下引用的原因，由于Http它是无状态的协议，所以每个http的请求都是独立的，这就导至了一个问题，我想要维持一个会话的话，需要将上次发送的请求在下一次请求中携带，这就导致了维持一次会话的代价很高，为了解决这个问题，引入了这个会话技术session和cookie

  • 其中cookie是客户端技术，它将客户的一些信息以键值对的形式，存储在浏览器的缓存中，存储的大小是由限度的，只能存储几kb，存储的类型是String
  • session是服务端技术，它存储是以键值对的形式，存储在服务器中，所以他的存储是受服务器存储容量限制，所以可以存放大量信息，存储的类型是，以object的类型存储在Map中

cookie和session的传输过程

  • 对于这个cookie，在每次发送请求的时候，会将存在浏览器缓存中的一些cookie信息，在请求头中携带（所以说它的存储大小是受限制的），服务器可以通过这个信息来确定是来自于那个客户端的吗，这样就可以保持这种对话，但由于cookie会在请求头中携带，所以安全性不高，很容易被人给抓包抓到，所以，一般不用来存储一些敏感的信息。
  • 存储一些敏感的信息，可以用session，session它其实是基于cookie的，会把该用户的信息存储到session中，并生成一个sessionID,返回给浏览器，cookie会存储这个sessionID 用来维持会话
  • “存储一些敏感的信息时，可以使用Session。我们举一个用户登录的例子来说一下session的作用过程，用户在浏览器输入用户名和密码，通过请求报文发送到服务器，服务器对去数据库中进行数据的查找，如果用户名和密码正确，Session会在用户成功登录后，将用户相关的数据存储在服务器端，并生成一个Session ID。服务器会将这个Session ID封装在响应报文中返回给浏览器。而这个Session ID 通常会被存储在Cookie中，以维持用户的会话状态。”
  • cookie它是可以跨引用的，存储cookie是使用的ConcrumentHashMap它是线程安全的，而sessions是不能够跨应用的，只能在本应用使用。如果session想要跨应用需要进行一些特殊的处理，比如将Session数据存储在持久化存储（如数据库）中，以实现可伸缩性和持久性。
  • Session数据通常在用户关闭浏览器后被删除，或者在一定时间内处于非活动状态后被删除。它们的持久性通常由服务器的配置来控制。而cookie的数据可以设置过期时间时间到了对cookie进行清楚，如果没有设置的话，关闭浏览器后数据就会被删除
